¿Que son las funciones?

Las funciones son un elemento central en el desarrollo. Cada función representa una unidad de procesamiento reutilizable que puede recibir uno o varios parámetros y devolver un valor. La escritura de funciones permite estructurar el código desacoplando de manera lógica las funcionalidades desarrolladas. Se recomienda, para una buena legibilidad y una buena mantenibilidad, limitar la longitud de las funciones.
Los objetivos de las funciones son:

  • Automatizar tareas repetitivas.
  • Mejorar la seguridad, confiabilidad y estabilidad del programa.
  • Encapsular o modularizar código, es decir, separarlo en tareas.

Aspectos Importantes

Los programas en C# como mínimo poseen una función, la función principal main(), esta función se llama desde el sistema operativo y no desde otra función. Las funciones no se pueden anidar, es decir, no puede haber una función dentro de otra. Se pueden tener o no argumentos para una función. En visual Basic una función es una serie de instrucciones dos segmentos Function y End Function. El Function procedimiento realiza una tarea y, a continuación, devuelve el control al código que realiza la llamada. Cuando devuelve el control, también devuelve un valor al código que realiza la llamada. Cada vez que se llama al procedimiento, se ejecutan sus instrucciones, empezando por la primera instrucción ejecutable después de la Function instrucción y finalizando con la primera End Functioninstrucción

Sintaxis

La sintaxis para declarar un Function procedimiento es la siguiente: 

            {{ CODECSHARP1 }}

La sintaxis general para escribir una función en C# es la siguiente:

            {{ CODECVB1 }}

Los modificadores pueden especificar el nivel de acceso y la información sobre la sobrecarga, invalidación, uso compartido y sombreado.

Declaración de funciones en C#

Cuando declaramos una función o método, no esta todo hecho, nos falta llamar a esa función para acceder utilizarla. Simplemente hay que poner el nombre de la función o método y poner entre paréntesis los parámetros que requieran. IMPORTANTE: en la llamada no hay que declarar las variables como en la declaración de la función, sino simplemente pasarles el valor.
